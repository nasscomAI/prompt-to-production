skills:
  - name: retrieve_policy
    description: Loads the raw text policy file and structures its contents into a mapped inventory strictly bound to explicitly numbered clauses.
    input:
      type: string
      format: A file path pointing to the text format policy document.
    output:
      type: list
      format: An array or dictionary mapping each numeric section identifier directly to its complete raw text content.
    error_handling: Halts and raises a specific flag rather than proceeding with partial data if the input document is structurally invalid or lacks identifiable numbered clauses, preventing initial clause omission.

  - name: summarize_policy
    description: Evaluates the structured clauses to generate a fully compliant summary where every condition and obligation natively references its exact source clause.
    input:
      type: list
      format: An array or dictionary of structured raw clauses generated by the retrieve_policy skill.
    output:
      type: string
      format: Formatted text containing the summary that explicitly preserves every condition, binding verb, and clause number.
    error_handling: To prevent obligation softening or condition drops (such as missing a required secondary approver), it enforces verbatim quotation and flagging if a multi-condition obligation cannot be summarized without meaning loss. To prevent scope bleed, it errors and refuses to insert phrases like 'standard practice' or assumptions not strictly present in the source document.
