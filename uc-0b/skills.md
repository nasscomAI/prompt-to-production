# skills.md

skills:
  - name: retrieve_policy
    description: Loads a plaintext policy file, parses its contents, and securely sections the text into a structured list of numbered clauses.
    input: A string representing the file path to the target .txt policy file.
    output: A dictionary or object containing structured representations of the clauses mapped by their respective clause numbers.
    error_handling: Handles missing file paths by throwing a FileNotFoundError, and flags unreadable text formats or empty documents with an immediate exception to halt downstream assumptions.

  - name: summarize_policy
    description: Receives the parsed structural sections and formulates a compliant summary that strictly enforces preservation of multi-conditions, obligation terminology, and explicit clause referencing.
    input: The structured object or dictionary of parsed clause sections generated by retrieve_policy.
    output: A meticulously formatted block of text delivering summarization items clearly referencing their source clause numbers and maintaining the exact structural obligations.
    error_handling: Automatically halts summarization of any distinct clause that risks verb softening or conditions dropping, instead appending a verbatim string of that clause with a prominent 'FLAG:' marker in the output.
