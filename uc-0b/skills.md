skills:
  - name: retrieve_policy
    description: Loads a plain text (.txt) policy file and parses its contents into structured, numbered sections.
    input: An absolute or relative file path pointing to the `.txt` policy document.
    output: A structured object (e.g., text, dictionary, or JSON array) where each numbered clause is isolated and accurately mapped to its clause number.
    error_handling: Refuse processing and return an explicit error string if the file is missing, empty, or lacks readable numbered clauses.

  - name: summarize_policy
    description: Consumes structured policy sections and produces a legally compliant summary preserving all clauses, conditions, and obligating verbs.
    input: The structured sections output generated by the `retrieve_policy` skill.
    output: A compiled summary text containing every clause reference, maintaining all explicitly stated requirements without any external assumptions.
    error_handling: Return a flagged error if a clause evaluates as unsummarizable without losing meaning, and output the problematic clause verbatim with a `NEEDS_REVIEW` flag.
