# skills.md — UC-0B: Summary That Changes Meaning

## Skill: faithful_policy_summary

---

### What This Skill Does

Reads a plain-text HR or policy document and produces a structured, clause-complete summary that preserves all obligations, entitlements, exceptions, and penalties — without changing meaning through omission, softening, or paraphrase.

---

### When to Use This Skill

- Input is a `.txt` policy document (HR leave, IT acceptable use, finance reimbursement, etc.)
- The goal is a human-readable summary that employees or managers can rely on
- Accuracy and completeness are mandatory — any meaning change is a defect

---

### Skill Steps (CRAFT Loop)

#### Step 1 — Catalogue
- Read the entire document top to bottom
- List every section heading and numbered/lettered clause
- Count total clauses (used later to verify completeness)

#### Step 2 — Risk-Flag
Before summarizing, identify:
- **Penalty clauses** (termination, suspension, financial penalty)
- **Numeric thresholds** (days, amounts, percentages, deadlines)
- **Conditional exceptions** ("unless", "except where", "provided that")
- **Conflicting clauses** (two rules that could contradict each other)

Mark each with a risk tag: `[PENALTY]`, `[NUMERIC]`, `[EXCEPTION]`, `[CONFLICT]`

#### Step 3 — Assemble Draft Summary
- Write one subsection per policy section
- Each numbered clause becomes one bullet point minimum
- Multi-part clauses become multiple bullets
- Preserve exact numbers, dates, thresholds, and conditional logic

#### Step 4 — Faithfulness Check (CRAFT Test)
Run the following checks before finalizing:

| Check | Question | Pass Condition |
|-------|----------|----------------|
| **Completeness** | Is every numbered clause covered? | Clause count in summary ≥ clause count in source |
| **Numeric Integrity** | Are all numbers preserved exactly? | No rounding, approximating, or omitting of figures |
| **Penalty Preservation** | Are all penalty clauses present and unambiguous? | No penalty softened to "may have consequences" |
| **Condition Integrity** | Are all "if/unless/except" conditions intact? | No conditional flattened to absolute rule |
| **No Invention** | Does the summary add anything not in the source? | Zero invented obligations or entitlements |

#### Step 5 — Format and Save
- Add "Critical Obligations" block at the very top (highest-stakes rules only)
- Output as `summary_[filename].txt`
- Append sign-off line: `[Summary generated by PolicySummarizer — verify against original before distribution]`

---

### Failure Modes to Avoid

| Failure | Example | Why It Matters |
|---------|---------|----------------|
| **Clause omission** | Dropping "Clause 4.3: No carry-forward of unused leave" | Employee falsely believes unused leave carries over |
| **Penalty softening** | "May result in action" instead of "Will result in termination" | Employee underestimates risk |
| **Numeric drift** | "About 10 days" instead of "12 days exactly" | Entitlement misrepresented |
| **Condition flattening** | "Medical cert required" omitting "only beyond 12 days" | Triggers unnecessary admin burden |
| **Cross-doc blending** | Adding a rule from another policy not in this document | Fabricates obligations that don't exist |

### Skill Parameters

| Parameter | Value |
|-----------|-------|
| Input format | `.txt` plain text |
| Output format | `.txt` structured summary |
| Output filename pattern | `summary_[source_filename].txt` |
| LLM temperature | 0 (deterministic, no creativity) |
| Max tokens | 2000 per document |
| Language | Match source document language |
