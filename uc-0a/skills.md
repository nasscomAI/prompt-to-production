# skills.md — UC-0A Complaint Classifier

skills:
  - name: classify_complaint
    description: >
      Classifies a single citizen complaint row into a structured output containing
      category, priority, reason, and flag by analyzing the complaint description text
      against the defined classification schema.
    input: >
      A single complaint record (dict or CSV row) with at minimum a 'description' field
      containing free-text citizen complaint text. Additional context fields (complaint_id,
      date_raised, city, ward, location, reported_by, days_open) may be present but are
      NOT used for classification — only 'description' drives the output.
    output: >
      A dict or CSV row with exactly four fields:
        - category (str): One of — Pothole, Flooding, Streetlight, Waste, Noise, Road Damage, Heritage Damage, Heat Hazard, Drain Blockage, Other
        - priority (str): One of — Urgent, Standard, Low
        - reason (str): One sentence citing specific words from the description that justify the category and priority
        - flag (str): "NEEDS_REVIEW" if the complaint is genuinely ambiguous between two or more categories, otherwise blank ("")
    error_handling: >
      If the 'description' field is missing or empty, return category: "Other", priority: "Standard",
      reason: "No description provided — unable to classify", flag: "NEEDS_REVIEW".
      If the description is too vague to map to any of the 9 specific categories with reasonable
      confidence, return category: "Other" with flag: "NEEDS_REVIEW" and cite the vague terms in
      the reason field. Never guess or hallucinate a category — default to "Other" with a flag.

  - name: batch_classify
    description: >
      Reads an input CSV file of citizen complaints, applies the classify_complaint skill
      to each row, and writes the results to an output CSV with the original columns plus
      the four classification columns (category, priority, reason, flag).
    input: >
      Two file paths:
        - input_path (str): Path to a CSV file containing complaint rows with columns
          [complaint_id, date_raised, city, ward, location, description, reported_by, days_open].
          The 'category' and 'priority_flag' columns are intentionally stripped — they must be
          generated by the classifier.
        - output_path (str): Path where the classified output CSV will be written.
    output: >
      A CSV file at output_path containing all original columns from the input CSV plus four
      appended columns: category, priority, reason, flag. Every row from the input must appear
      in the output — no rows may be dropped or duplicated. Column order is preserved with the
      four classification columns appended at the end.
    error_handling: >
      If the input file does not exist or is not a valid CSV, raise a clear error message with
      the file path and exit without writing any output. If individual rows fail classification
      (e.g., missing description), classify them with category: "Other", priority: "Standard",
      reason: "Classification failed — [specific reason]", flag: "NEEDS_REVIEW" and continue
      processing remaining rows. Never silently skip rows. Log a summary at the end: total rows
      processed, successful classifications, and rows flagged as NEEDS_REVIEW.
