# skills.md — UC-0B: Summary That Changes Meaning

## Skill: Policy Document Summarisation with Completeness Guarantee

### What this skill does
Transforms a full-length HR or Finance policy document into a structured
summary that:
- Maps every numbered clause to a corresponding numbered summary point
- Preserves all thresholds, limits, deadlines, and penalties verbatim
- Self-critiques and iteratively refines until no omission remains

---

### Trigger Conditions
Invoke this skill when:
- The input is a numbered/structured policy or compliance document
- The output will be read by employees who may not read the original
- Legal or operational consequences follow from misreading the document

---

### Skill Steps

#### Step 1 — Parse
```
Extract all numbered clauses from the document.
Return: List[(clause_number, clause_title, clause_body)]
```

#### Step 2 — Draft Summary
```
For each clause in the list:
  Write a 1–3 sentence summary that:
    - Starts with the clause number and title
    - Includes all specific values (days, amounts, percentages)
    - Uses the same consequence language as the original
```

#### Step 3 — Completeness Critique
```
Prompt:
  "Compare each numbered clause in the ORIGINAL to each numbered point
   in the SUMMARY.
   For each clause, answer:
     a) Is it present? (Yes / No)
     b) Are all numbers/thresholds preserved? (Yes / No / Partial)
     c) Is punitive language preserved? (Yes / No / Partial)
   List every 'No' or 'Partial' as a finding.
   End with VERDICT: COMPLETE or VERDICT: INCOMPLETE."
```

#### Step 4 — Refine
```
If VERDICT = INCOMPLETE:
  For each finding:
    Rewrite the affected summary point to fix the finding.
  Repeat Step 3.
  Maximum 3 iterations.
```

#### Step 5 — Final Output
```
Format:
  SUMMARY — {POLICY NAME}
  ========================
  1. {Clause Title}: {Summary sentence(s)}
  2. ...
  ...
  N. ...

  [Generated by UC-0B CRAFT loop · {timestamp}]
```

---

### Key Heuristics

| Heuristic | Rule |
|-----------|------|
| Number preservation | Any digit in the original clause must appear in the summary point |
| Penalty preservation | Words like "termination", "LWP", "legal action", "lapse" must not be replaced with softer synonyms |
| Clause count | `len(summary_points) >= len(original_clauses)` |
| Merge prevention | One clause → one summary point (never merge two clauses) |

### Prompt Templates

#### Initial Summary Prompt
```
You are a precise policy analyst.
Summarise the following policy document.
Include EVERY numbered clause as a separate numbered point.
Preserve all numbers, thresholds, deadlines, and penalties exactly.
Do not soften punitive language.

DOCUMENT:
{policy_text}

Output a numbered summary matching the clause structure above.
```

#### Critique Prompt
```
ORIGINAL DOCUMENT:
{policy_text}

SUMMARY TO CRITIQUE:
{summary}

TASK:
List every clause from the original that is:
  a) Missing entirely from the summary
  b) Present but with key numbers/thresholds changed or omitted
  c) Present but softened so the obligation is unclear

End with:
VERDICT: COMPLETE   (if no issues found)
VERDICT: INCOMPLETE (if any issues found)
```

#### Refine Prompt
```
ORIGINAL DOCUMENT:
{policy_text}

PREVIOUS SUMMARY:
{previous_summary}

CRITIQUE:
{critique}

TASK:
Produce a corrected summary that fixes every finding in the critique.
Rules:
  1. Every numbered clause must appear as a numbered point.
  2. All thresholds, limits, and numbers must be exact.
  3. No clause may be softened, merged, or omitted.
  4. Use plain language but preserve legal precision.

Output the refined summary only, no preamble.
```

### Known Failure Patterns This Skill Fixes

| Pattern | Example of Bad Output | How Skill Fixes It |
|---------|-----------------------|--------------------|
| Clause omission | Carryover limit (Clause 3) not mentioned | Step 3 critique flags it; Step 4 inserts it |
| Threshold loss | "a few days" instead of "3 consecutive days" | Number preservation heuristic catches it |
| Penalty softening | "may lead to action" vs "up to termination" | Punitive language check in critique |
| Silent encashment cap | Omits "300 days total" cap | Digit check requires all numbers present |
